Declaración de Accesibilidad

Compromiso de accesibilidad

Nos comprometemos a garantizar que nuestro sitio web, Diccionario Médico, sea accesible para todas las personas, incluidas aquellas con discapacidades. Nos basamos en las Pautas de Accesibilidad para el Contenido Web (WCAG) 2.2 en nivel AA, según el Real Decreto 1112/2018, para asegurar el cumplimiento de los requisitos de accesibilidad.

Características de Accesibilidad Implementadas

  • Tablas Accesibles:
    Criterio de conformidad: 1.3.1 Info and Relationships (A)
    Todas las tablas están correctamente etiquetadas usando las etiquetas <thead>, <tbody>, y <caption> para asegurar que los usuarios de tecnologías asistivas comprendan la estructura de los datos.
  • Uso de color:
    Criterio de conformidad: 1.4.1 Use of Color (A)
    El color no es el único medio para transmitir información. Las indicaciones o acciones importantes se muestran mediante combinaciones de colores y estilos de texto (por ejemplo, subrayados o negritas) que son claras y accesibles para usuarios con deficiencias visuales o daltónicos.
  • Contraste:
    Criterio de conformidad: 1.4.3 Contrast (Mínimo) (AA)
    El color del texto y del fondo tiene un contraste mínimo de 4.5:1 para el texto normal y 3:1 para el texto grande. En la página web, los colores elegidos entre el fondo y el texto aseguran una adecuada legibilidad para los usuarios con baja visión o daltonismo. El CSS aplicado permite mantener este contraste en todas las páginas del diccionario.
  • Escalabilidad del texto:
    Criterio de conformidad: 1.4.4 Resize Text (AA)
    El sitio web permite escalar el texto mediante las funcionalidades del navegador sin que el diseño se rompa. Los estilos CSS aplicados garantizan que al aumentar el tamaño del texto hasta un 200%, los contenidos siguen siendo legibles y utilizables sin pérdida de funcionalidad.
  • Espaciado de texto:
    Criterio de conformidad: 1.4.12 Text Spacing (AA)
    Se asegura un espaciado adecuado entre líneas y párrafos para mejorar la legibilidad del contenido.
  • Foco visible:
    Criterio de conformidad: 2.4.7 Focus Visible (AA)
    Cuando un elemento interactivo (como un enlace o un botón) es seleccionado o navegado mediante teclado, el CSS asegura que se visualice un borde visible o un cambio de color, facilitando la navegación a los usuarios que utilizan tecnologías asistivas o que prefieren utilizar el teclado.
  • Orden de Foco Secuencial:
    Criterio de conformidad: 2.4.3 Focus Order (A)
    Se ha implementado un orden de foco lógico y secuencial en todos los elementos interactivos, lo que permite navegar a través del contenido de manera predecible.
  • Propósito de los Enlaces Claro:
    Criterio de conformidad: 2.4.4 Link Purpose (In Context) (A)
    El propósito de cada enlace es claramente identificable por su texto y contexto, para facilitar la comprensión a los usuarios. Ejemplo: Volver al inicio.
  • Mapas del sitio:
    Criterio de conformidad: 2.4.5 Multiple Ways (AA)
    Se ha implementado un mapa del sitio para mejorar la navegación.
  • Formularios Accesibles:
    Criterio de conformidad: 3.3.2 Labels or Instructions (A), 1.3.5 Identify Input Purpose (AA)
    Los formularios en el sitio web están correctamente etiquetados mediante la etiqueta <label> asociada a los controles, y se han proporcionado instrucciones claras para el uso de los mismos.
  • Encabezados Jerárquicos Correctamente Implementados:
    Criterio de conformidad: 2.4.6 Headings and Labels (AA) y 2.4.10 Section Headings (AAA)
    Se ha respetado la jerarquía correcta de los encabezados (<h1>, <h2>, <h3>) en las páginas, lo que facilita la navegación para los usuarios que dependen de la estructura de la página para entender el contenido.
  • Saltar a contenido:
    Criterio de conformidad: 2.4.1 Bypass Blocks (A)
    El enlace "saltar a contenido" está presente en todas las páginas del sitio, permitiendo a los usuarios saltar directamente al contenido principal sin tener que pasar por todos los menús de navegación.
  • Texto no justificado:
    Criterio de conformidad: 1.4.8 Visual Presentation (AAA)
    En las hojas de estilo accesibles, el texto no está justificado para mejorar la legibilidad, facilitando la lectura para todos los usuarios.
